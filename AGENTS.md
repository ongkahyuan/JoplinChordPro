# AGENTS.md
## Build
- `npm run dist` — builds the plugin. Runs 3 webpack steps in sequence:
  1. `buildMain` — compiles `src/index.ts`, copies non-TS assets from `src/` to `dist/`
  2. `buildExtraScripts` — compiles any scripts listed in `plugin.config.json` `extraScripts`
  3. `createArchive` — packages `dist/` into `publish/com.chordpro.renderer.jpl` + `.json` info file
- Output lands in `dist/` (compiled) and `publish/` (distributable `.jpl` archive). Both are gitignored.
## Source layout
- `src/index.ts` — plugin entry point (registers `onStart` hook via `joplin.plugins.register`)
- `src/manifest.json` — plugin metadata (id, version, name). Version must stay in sync with `package.json`
- `api/` — Joplin plugin API type declarations. Imported as `import joplin from 'api'` (webpack alias, not an npm package)
## No test/lint setup
No test runner, linter, or formatter is configured. Do not assume jest, eslint, or prettier exists.
## Version bumping
- `npm run updateVersion` — increments the patch version in both `package.json` and `src/manifest.json`
## External/content scripts
To add a content script or webview script that needs compilation:
1. Place the file under `src/` (e.g. `src/webviews/index.ts`)
2. Add its path relative to `src/` to `extraScripts` in `plugin.config.json`
3. Compiled output always gets `.js` extension; reference it as such
## Publishing
- `npm publish` — the `publish/` dir (containing `.jpl` + `.json`) is what gets distributed
- Joplin plugin repo auto-picks it up if: package name starts with `joplin-plugin-`, keywords include `joplin-plugin`, and `publish/` contains `.jpl` + `.json`
## Webpack config caveat
`webpack.config.js` is generated by `generator-joplin` and overwritten on `npm run update`. Custom logic should go in a separate JS file and be required in, not edited inline. See `plugin.config.json` `webpackOverrides` for the supported extension point.
